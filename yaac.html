YAAC (Yet Another APRS Client)

YAAC is a cross-platform APRS client written in Java, developed by Andrew Pavlin (KA2DDO). It provides full APRS mapping, messaging, and tracking capabilities.

Overview

YAAC is a feature-rich APRS application that runs on any platform with Java. It supports multiple TNC connections, online and offline mapping, and comprehensive APRS messaging.

Key Features

  • Cross-platform: Windows, Linux, macOS
  • Multiple TNCs: Connect to several radios
  • Offline maps: OpenStreetMap tiles
  • GPS support: Real-time tracking
  • Messaging: Full APRS messaging
  • Digipeater: Fill-in digi support
  • Weather: WX station support

Starting YAAC

# From command line
java -jar /opt/yaac/YAAC.jar

# Or use desktop launcher

Initial Configuration

Station Setup

  1. File → Configure → Station Identification
  2. Enter callsign with SSID (e.g., VA2OPS-9)
  3. Set position or enable GPS
  4. Choose icon (car, house, etc.)

TNC/Modem Connection

File → Configure → Add Port:

TypeKISS-over-TCP
Hostlocalhost

Offline Maps

YAAC works with EmComm-Tools offline map tiles:

  1. File → Configure → Maps
  2. Add tile source from local mbtiles
  3. Or download tiles for your area

Sending Messages

  1. Click on a station on the map
  2. Or use Messages → New Message
  3. Enter destination callsign
  4. Type message (max 67 characters)
  5. Click Send

Beaconing

Configure → Beacon Settings:

  • Fixed: Set interval (e.g., 30 min for home)
  • SmartBeaconing: Speed-based (mobile)
  • Status text: Shown with your beacon

Digipeater Setup

For fill-in digipeating:

  1. Configure → Digipeater
  2. Enable digipeating
  3. Set alias (typically WIDE1-1)
  4. Configure duplicate filtering

Troubleshooting

No Stations Visible

  • Check TNC connection status
  • Verify frequency (144.390 in NA)
  • Add APRS-IS port for testing

Map Not Loading

  • Check internet for tile download
  • Configure offline tiles
  • Verify map source settings
