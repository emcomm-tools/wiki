ARDOP (Amateur Radio Digital Open Protocol)

ARDOP is an open-source sound card modem for amateur radio, providing reliable HF data communications. It's commonly used with Winlink and is fully free/open source.

Overview

ARDOP provides ARQ (Automatic Repeat reQuest) data transfer over HF radio. While slower than VARA, it's completely free and open source, making it an important option for amateur radio data communications.

Key Features

  • Open source: Free to use, modify, study
  • ARQ protocol: Guaranteed delivery
  • Adaptive: Adjusts to conditions
  • Cross-platform: Linux, Windows, Raspberry Pi

ardopcf (Linux Version)

EmComm-Tools includes ardopcf, an improved fork:

  • Better performance than original ARDOP
  • Native Linux support
  • Active development

Running ARDOP

# Basic start
ardopcf 8515 plughw:1,0 plughw:1,0

# Arguments:
# 8515 = TCP port for client connections
# plughw:1,0 = capture device
# plughw:1,0 = playback device

With PTT

# Using serial RTS for PTT
ardopcf 8515 plughw:1,0 plughw:1,0 -p /dev/ttyUSB0

# Using hamlib
ardopcf 8515 plughw:1,0 plughw:1,0 -c 1

Pat Winlink Configuration

In Pat's config.json:

"ardop": {
    "addr": "localhost:8515",
    "arq_bandwidth": {
        "Forced": false,
        "Max": 2000
    },
    "rig": "",
    "ptt_ctrl": true
}

Connect to gateway:

pat connect ardop:///WL2K-GATEWAY

Finding Your Audio Device

# List capture devices
arecord -l

# List playback devices  
aplay -l

# Example output:
# card 1: Device [USB Audio Device], device 0: USB Audio
# Use: plughw:1,0

Troubleshooting

No Audio

  • Verify device names with arecord/aplay
  • Check alsamixer levels
  • Test audio path first

Connection Fails

  • Check frequency/gateway
  • Verify PTT is working
  • Ensure adequate signal strength

Slow Speeds

  • ARDOP adapts to conditions
  • Try wider bandwidth setting
  • Propagation affects speed
