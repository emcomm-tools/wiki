JS8Call

JS8Call is a weak-signal digital mode for keyboard-to-keyboard messaging, developed by Jordan Sherer (KN4CRD). Based on FT8, it's optimized for conversational QSOs and store-and-forward messaging, making it ideal for emergency communications.

Overview

JS8Call combines FT8's excellent weak-signal performance with the ability to have real conversations. Unlike FT8 which is designed for quick exchanges, JS8Call supports free-form text messaging, automatic relaying, and network-style operations.

Key Features

  • Weak signal: Works at -24 dB SNR
  • Keyboard-to-keyboard: Type and send any message
  • Store and forward: Relay messages through other stations
  • Heartbeat network: See who's on the air
  • APRS gateway: Send position/messages to APRS
  • Automatic relay: Extend range via other stations
  • Multiple speeds: Trade speed for reliability

Configuration

Station Setup

  1. Go to File → Settings → General
  2. Enter your callsign
  3. Enter your grid square (4 or 6 char)
  4. Set station info message (optional)

Audio Setup

  1. Go to File → Settings → Audio
  2. Select your sound card interface
  3. For Digirig: "USB Audio CODEC"

Radio/CAT Control

  1. Go to File → Settings → Radio
  2. Select your rig model
  3. Set serial port (/dev/ttyUSB0 or /dev/digirig)
  4. Configure baud rate
  5. Set PTT method (CAT recommended)

Common Commands

CommandDescription

Making a Contact

  1. Set audio levels (use Tune button)
  2. Select clear offset in waterfall
  3. Send @ALLCALL CQ
  4. Wait for responses in activity window
  5. Double-click callsign to reply
  6. Type your message, press Enter
  7. End with 73 and SK

EmComm Features

  • Store & Forward: Messages relay through other stations
  • Automatic Relay: Extend network coverage
  • Heartbeat Network: Situational awareness
  • APRS Gateway: Position and message integration
  • Group Calls: @GROUP for team messaging

Troubleshooting

No Decode

  • Check audio input levels (30-50 dB)
  • Verify correct audio device
  • Confirm radio is in USB mode

No Transmit

  • Test CAT control in settings
  • Verify PTT method
  • Check audio output device
