Winlink Catalog & Inquiry Requests

Winlink provides automated information services accessible via radio email. These "catalog" or "inquiry" requests allow operators to retrieve weather forecasts, propagation data, position reports, GRIB files, and system information—critical resources when internet is unavailable.

Overview

Winlink's Query Catalog system provides automated information retrieval via specially formatted email messages. When connected to an RMS gateway, you send a request message, and the system responds with the requested information. This works entirely over radio—no internet required at your end.

How It Works

  1. Compose an inquiry message with specific formatting
  2. Connect to an RMS gateway and send the message
  3. Disconnect and wait a few minutes
  4. Reconnect to receive the response

Using Winlink Express (Windows)

Accessing the Catalog

  1. Click Settings → Winlink Catalog Request
  2. Click Update via Internet (if available) to refresh the catalog list
  3. Select a category from the left panel
  4. Double-click items to add to your request list
  5. Click Post Request to create the message
  6. Connect to an RMS and send/receive

Manual Catalog Request

You can also compose requests manually:

To: INQUIRY@winlink.org
Subject: Catalog Request
Body:
<category> <item>

Using Pat (Linux)

With the Pat Winlink client, compose messages to the catalog addresses:

# Create a catalog request
echo "WX_US FORECAST_CA" | pat compose INQUIRY@winlink.org -s "Catalog"

# Or use patmail.sh script
echo "WX_US FORECAST_CA" | patmail.sh INQUIRY "Catalog" telnet

SailDocs Email Requests

SailDocs is an automated email service providing weather data, optimized for low-bandwidth connections like Winlink.

Basic SailDocs Request

To: query@saildocs.com
Subject: (anything)
Body:
send info
send home

Requesting Web Pages

SailDocs can fetch and convert web pages to text:

To: query@saildocs.com
Subject: Weather Request
Body:
send https://tgftp.nws.noaa.gov/data/forecasts/zone/ca/caz006.txt

Hurricane Information

To: query@saildocs.com
Subject: Hurricane Info
Body:
send https://tgftp.nws.noaa.gov/data/hurricane_products/atlantic/weather/discussion.txt
send https://tgftp.nws.noaa.gov/data/hurricane_products/atlantic/weather/outlook.txt
send https://www.nhc.noaa.gov/text/MIATCMAT1.shtml

GRIB Weather Files

GRIB (GRIdded Binary) files contain weather model data that can be displayed on chart plotters and navigation software.

GRIB Request Syntax

To: query@saildocs.com
Subject: GRIB
Body:
send gfs:LAT1,LAT2,LON1,LON2|GRID|TIMES|PARAMS

Parameters

  • LAT1,LAT2: Latitude bounds (e.g., 30N,50N)
  • LON1,LON2: Longitude bounds (e.g., 130W,110W)
  • GRID: Grid spacing in degrees (e.g., 1,1 or 2,2)
  • TIMES: Forecast hours (e.g., 0,24,48,72)
  • PARAMS: Data types (WIND, PRESS, WAVES, RAIN, etc.)

GRIB Examples

# Basic wind and pressure for US West Coast
send gfs:30N,50N,130W,110W|2,2|24,48,72|WIND,PRESS

# Detailed forecast with waves
send gfs:25N,45N,140W,115W|1,1|0,12,24,36,48,72|WIND,WAVES,PRESS

# Atlantic crossing forecast
send gfs:30N,45N,70W,10W|2,2|0,24,48,72,96,120|WIND,PRESS,WAVES

GRIB Subscription

To receive daily updates, change "send" to "sub":

sub gfs:30N,50N,130W,110W|2,2|0,24,48,72|WIND DAYS=7

Cancel with the same request using "cancel" instead of "sub".

Available GRIB Models

Send blank email to gribmodels@saildocs.com for complete list:

  • gfs: - NOAA GFS global model (default)
  • nam: - North American Mesoscale
  • coamps: - Navy COAMPS model
  • ww3: - WaveWatch III (waves only)

Position Reports

Sending Position Reports

Position reports can be sent automatically (GPS) or manually through Winlink Express:

  1. Settings → GPS/Position Reports
  2. Configure GPS COM port or enter manual coordinates
  3. Click "Send Position Report"

Requesting Position Reports

To request another station's position:

To: INQUIRY@winlink.org
Subject: Position Request
Body:
QTH CALLSIGN

Position Report Destinations

Position reports are automatically sent to:

  • Winlink position map (winlink.org/userPositions)
  • APRS-IS (appears on aprs.fi)
  • ShipTrak and YotReps maritime tracking

NWS Weather by Email

Direct NWS Zone Forecasts

Get forecasts directly from NOAA/NWS:

To: query@saildocs.com
Subject: NWS Forecast
Body:
send https://tgftp.nws.noaa.gov/data/forecasts/zone/ca/caz006.txt

Finding Your Zone Code

  1. Visit: https://tgftp.nws.noaa.gov/data/forecasts/zone/
  2. Select your state (2-letter code)
  3. Find your county/zone file (e.g., caz006.txt for California zone 6)

Alternative Weather Service

Third-party weather by email (requires Winlink account):

To: weather@hamdesk.com
Subject: 97103

(Returns short forecast for ZIP code 97103)

For detailed forecast, use "full" prefix:

Subject: full 97103

Other Email Services

Spot Forecasts

Get point forecast for specific coordinates:

To: query@saildocs.com
Subject: Spot
Body:
send spot:45.5N,122.7W|7,12|

Returns 7-day forecast at 12-hour intervals for Portland, OR area.

Buoy Weather Data

To: query@saildocs.com
Subject: Buoy
Body:
send http://www.ndbc.noaa.gov/data/latest_obs/46029.txt

//WL2K R/Weather request for deployment
//WL2K P/Resource status update needed
//WL2K Z/Medical emergency - need airlift coordinates

Tips for Efficient Catalog Use

  • Minimize data: Request only what you need—airtime is limited
  • Use subscriptions: For daily weather, subscriptions save connection time
  • Batch requests: Combine multiple requests in one message
  • Check file sizes: GRIB files can be large; use coarser grids (2,2 vs 1,1)
  • Time zones: Weather products use UTC/Zulu time
  • Update catalog: Refresh the catalog periodically for new services
