ICS Forms for Amateur Radio EmComm

The Incident Command System (ICS) uses standardized forms for emergency communications. Amateur radio operators supporting served agencies should be familiar with these forms.

Common ICS Forms

ICS-213: General Message

The most frequently used form for sending messages:

FieldDescription
ToRecipient name/position
FromSender name/position
SubjectBrief topic description
Date/TimeWhen message was created
MessageBody of the message
Approved ByAuthorizing official
ReplySpace for response

ICS-214: Activity Log

Record of activities and communications:

  • Track all radio traffic
  • Document events chronologically
  • Record personnel assignments

ICS-205: Incident Radio Communications Plan

Lists all radio frequencies and assignments:

  • Channel assignments
  • Frequencies (TX/RX)
  • Tone/NAC codes
  • Net designations

ICS-309: Communications Log

Detailed log of radio transmissions:

  • Time of each transmission
  • From/To stations
  • Message summary
  • Operator initials

Sending Forms via Radio

Using Flmsg (NBEMS)

  1. Open Flmsg
  2. Select Form → ICS-213
  3. Fill in all fields
  4. Click AutoSend to transmit via Fldigi

Using Winlink

  1. Open Pat or Winlink Express
  2. Compose → Templates → ICS Forms
  3. Complete the form
  4. Send via radio gateway

Radiogram Format

Standard ARRL message format:

Preamble

  • Number: Message sequence number
  • Precedence: R, W, P, or EMERGENCY
  • Handling: HXA-HXG codes
  • Station of Origin: Originating callsign
  • Check: Word count
  • Place of Origin: City, State
  • Time Filed: Zulu time
  • Date: Message date

Address

Complete recipient information including phone if available.

Text

Message body - count words carefully.

Signature

Sender's name.

Prowords

ProwordMeaning
BREAKSeparates parts of message
FIGURESNumbers follow
I SPELLPhonetic spelling follows
INITIALSingle letter follows
MIXED GROUPLetters and numbers mixed
CORRECTIONError, re-sending
ROGERMessage received correctly
SAY AGAINPlease repeat
OVEREnd of transmission, reply expected
OUTEnd of contact

Best Practices

  1. Print clearly - Forms may be photocopied
  2. Use 24-hour time - Avoid AM/PM confusion
  3. Use Zulu (UTC) - Standard for all logs
  4. Be concise - Keep messages short
  5. Proofread - Errors waste time
  6. Keep copies - Document everything
