What is JS8Call?

JS8Call is a weak-signal digital mode based on FT8's modulation, but designed for actual conversations and messaging rather than just quick signal reports. It's optimized for:

  • Keyboard-to-keyboard chat - Real conversations
  • Store and forward messaging - Messages held for offline stations
  • Automatic relaying - Multi-hop message delivery
  • Weak signal performance - Works at -24dB SNR
  • Automated queries - Grid, info, status requests
  • Group calls - @ALLCALL, custom groups

Speed Modes

Mode Speed Frame SNR Threshold Best Use
Slow ~8 WPM 30 sec -28 dB Extreme weak signal, LF/MF

Fast ~24 WPM 10 sec -20 dB Good conditions, local

Speed Selection: Start with Normal. Only use Slow when signals are extremely weak. Fast/Turbo require stronger signals but allow quicker exchanges.

Directed Commands

JS8Call supports special commands sent to other stations:

Information Queries

  • SNR? - Request signal report
  • INFO? - Request station info
  • GRID? - Request grid square
  • STATUS? - Request status message
  • HEARING? - Who are they hearing?

Network Commands

  • QUERY CALL [call]? - Have you heard [call]?
  • RELAY [call] [msg] - Relay message
  • CMD [command] - Send command
  • MSG [text] - Store message
  • MSG TO:[call] [text] - Message for relay

Responses

  • SNR [value] - Signal report
  • INFO [text] - Station info
  • GRID [grid] - Grid square
  • STATUS [text] - Status message
  • ACK - Acknowledge

Special

  • HB - Heartbeat
  • CQ CQ CQ - General call
  • AGN? - Say again
  • QSL - Confirmed
  • 73 - End of contact

Command Examples

// Requesting signal report from W1ABC VA2OPS: W1ABC SNR? // W1ABC's response W1ABC: VA2OPS SNR -15 // Querying if W1ABC has heard W2XYZ VA2OPS: W1ABC QUERY CALL W2XYZ? // Sending free-form message VA2OPS: W1ABC HOW COPY? QTH GRANBY QC

Messaging

üì¨ JS8Call Inbox/Outbox

JS8Call has built-in store-and-forward messaging:

  • Messages stored locally until delivered
  • Automatic retry when station heard
  • Works even when recipient offline
  • Messages can be relayed via other stations

Sending a Stored Message

  1. Open File ‚Üí Show Message Inbox
  2. Click Compose
  3. Enter destination callsign
  4. Type message (keep it short!)
  5. Click Send Later (queued) or Send Now

Retrieving Messages for You

When you come on the air, other stations may have messages stored for you:

// Query all hearing stations for messages @ALLCALL QUERY MSGS? // Stations with messages respond with MSG count

Enabling Relay

In JS8Call settings (File ‚Üí Settings ‚Üí General):

  • Enable Auto-Reply - Respond to queries
  • Enable Relay - Forward messages for others
  • Store Messages - Hold messages for offline stations
Be a Good Citizen: Enable relay and message storage if you can monitor the frequency. This builds the network and helps messages get delivered.

Groups (@ALLCALL, etc.)

JS8Call supports group calls using @ prefix:

GroupMeaningUse
@ALLCALL All stations General queries, announcements
@HB Heartbeat group Network presence
@APRSIS APRS-IS gateway APRS position reports
@GROUPNAME Custom group EmComm nets, clubs, etc.

Creating Custom Groups

  1. Go to File ‚Üí Settings ‚Üí General
  2. In "Groups" field, add your groups (comma-separated)
  3. Example: @ARES,@EMCOMM,@VA2GCU
  4. Now you'll respond to calls to those groups
// Calling all ARES members VA2OPS: @ARES NET CHECKIN - QRV FOR TRAFFIC // All stations subscribed to @ARES will decode this

Heartbeat Network

üíì Heartbeat (HB)

Automatic beacon that announces your presence:

  • Sends brief transmission every few minutes
  • Includes your grid square
  • Lets others know you're monitoring
  • Builds network map automatically

Enabling Heartbeat

  1. File ‚Üí Settings ‚Üí General
  2. Check Enable Heartbeat Networking
  3. Set interval (10-30 min typical)
  4. Set sub-modes (Normal, Fast, etc.)

Heartbeat Acknowledgment

When you see another station's heartbeat, you can:

  • HB ACK - Acknowledge their heartbeat
  • This tells them you can hear them
  • Builds bi-directional path knowledge

EmComm Operations

üö® JS8Call for Emergency Communications

JS8Call excels for EmComm because:

  • Works at very weak signal levels
  • Store-and-forward bridges timing gaps
  • Relay extends range beyond direct propagation
  • Groups allow targeted announcements
  • Automatic queries reduce coordination overhead

EmComm Setup Recommendations

  • Create local group: @YOURARESGROUP, @YOURCLUB
  • Enable relay and message storage
  • Set meaningful INFO: "VA2OPS John - Granby QC - EmComm"
  • Set STATUS: "QRV for traffic" or current situation
  • Enable heartbeat so others know you're monitoring

Emergency Checkin Example

// Net control opens net W1NCS: @ARES ARES NET OPEN - CHECK IN WITH STATUS // Station checks in VA2OPS: W1NCS QRV PWR OK SHELTER 3 // NCS acknowledges W1NCS: VA2OPS RR SHELTER 3 LOGGED

Sending Traffic via JS8Call

// Direct message to specific station W1EOC: MSG SHELTER 3 NEEDS 50 BLANKETS - POPULATION 127 // Message via relay W1ABC: MSG TO:W1EOC SHELTER 3 NEEDS 50 BLANKETS DE VA2OPS // Query if message was delivered @ALLCALL: QUERY CALL W1EOC?

Operational Tips

üì° Best Practices

  • Keep messages SHORT
  • Use standard abbreviations
  • Enable relay when monitoring
  • Use appropriate speed mode
  • Set accurate grid square

‚öôÔ∏è Settings

  • Use correct audio levels
  • Enable heartbeat (10-30 min)
  • Join relevant groups
  • Update INFO and STATUS

üîß Troubleshooting

  • No decodes? Check audio levels
  • Time must be accurate (¬±2 sec)
  • USB mode only (not LSB)
  • Verify dial frequency
  • Check waterfall for signals

üìù Abbreviations

  • QRV = Ready
  • QRT = Signing off
  • QSL = Confirmed
  • RR = Roger
  • DE = From
Time Sync Critical: JS8Call requires accurate computer time. Sync to internet time server or GPS. Even 2-3 seconds off causes decode problems.
