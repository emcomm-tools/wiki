What is APRS?

APRS (Automatic Packet Reporting System) is a digital communications system that uses packet radio to share real-time data including:

  • Position/Location - GPS coordinates on a map
  • Messages - Short text messages between stations
  • Weather Data - Temperature, wind, rain, barometric pressure
  • Telemetry - Sensor data, status information
  • Objects/Items - Mark locations on the map
  • Bulletins - Announcements to all stations

APRS was developed by Bob Bruninga, WB4APR (SK), and has become a worldwide network for position tracking and tactical communications.

Digipeater Paths

Important: Using the wrong path can cause network congestion. Use the minimum path needed to reach your destination!

Recommended Paths

PathUse CaseHops
WIDE1-1 Local area, fill-in digis 1
WIDE1-1,WIDE2-1 Standard mobile path (recommended) 2
WIDE2-1 Good signal, skip fill-ins 1
WIDE2-2 Rural areas, limited digis 2
(none) Direct to IGate, no digipeating 0
Golden Rule: Use WIDE1-1,WIDE2-1 for mobile. It provides good coverage without flooding the network. Never use WIDE paths greater than 2-2 total.

Paths to AVOID

  • RELAY,WIDE - Obsolete, causes problems
  • WIDE3-3 or higher - Network abuse
  • WIDE7-7 - Definitely don't do this!

APRS Messaging

APRS supports two-way text messaging between stations:

VA2OPS-9>APRS::W1ABC :Meeting at shelter 3 at 1400{123

Message Features

  • Addressed messages - Sent to specific callsign
  • Acknowledgment - Recipient confirms receipt
  • Retry - Automatic retransmission until ack'd
  • Message number - Tracks messages ({123 in example)
  • Max length - ~67 characters

Special Message Addresses

APRS for EmComm

EmComm Applications: APRS is extremely valuable for emergency communications - it provides real-time situational awareness that voice communications cannot match.

EmComm Uses

ApplicationHow APRS Helps
Asset Tracking Track vehicles, teams, resources on map in real-time
Route Planning See where units are, optimize dispatch
Weather Reporting Weather stations feed current conditions
Object Placement Mark shelters, incidents, hazards on map
Messaging Text messages don't tie up voice channels
Status Reporting Automatic position + status updates

Tactical APRS Setup

  • Use consistent SSIDs across your team
  • Pre-program symbols for different unit types
  • Use objects to mark fixed locations (shelters, EOC, etc.)
  • Consider local digipeater if coverage is poor
  • Have an IGate to bridge RF to internet for wide visibility
Object Example: Place an object called "SHELTER1" at the shelter location so everyone can see it on the map, even if no one is transmitting from there yet.

Quick Setup Checklist

  1. Program radio to 144.390 MHz (North America)
  2. Set callsign with appropriate SSID (-9 for mobile)
  3. Configure path: WIDE1-1,WIDE2-1
  4. Select appropriate symbol
  5. Set reasonable beacon rate
  6. Enter status text/comment
  7. Verify GPS lock before transmitting
  8. Check aprs.fi to confirm you're being heard
